二、进阶

（一）面向对象

1.类的继承

  • A extends B
    • A继承自B
    • A是子类、B是父类
    • 子类拥有父类的所有成员
    • 子类可以重写父类成员
    • this指向是动态的
    • super指向父类
  • A instanceof B
    • A是不是一个B
    • 子类对象始终可以赋值给父类对象，当只能使用父类定义的成员，应使用类型保护
    • 单根性：每个类只有一个父类（避免多继承）
    • 传递性：隔代也是父子
  • 修饰符
    • public 均可以访问
    • protected 自身及子类可以访问
    • private 自身可访问

2.抽象类

  • abstract
    • 某个类只表示一个抽象的概念，只用于提取子类共有的成员，而不能直接创建对象
    • 抽象成员，父类只知道类型，不知道具体的实现，且子类必须要将其实现
  • 模板模式
    • 所有子类实现某个方法流程一致，只有某一步骤不一致
      • 可能出现大量重复代码
      • 重复代码提取后，可能出现调用顺序紊乱
    • 将该方法提取到父类实现
    • 将不一致的地方提取到父类为抽象方法，让子类去具体实现
    • 在父类方法中使用抽象方法

3.静态成员

  • static
    • 某个成员属于类本身，而不是具体的对象
    • 静态方法中this指向当前类
  • 单例模式
    • 某些类的实例对象，在系统中最多只能有一个（避免随意创建）
    • 将构造函数私有 private
    • 定义一个私有的静态属性，用于存储实例对象
    • 定义一个公有的静态方法，用于得到实例对象
      • 判断静态属性是否已经有值，有值则直接返回该实例
      • 没有则创建一个实例并赋值给静态属性，然后返回该实例

4.接口

  • implements
    • 表达某个类是否拥有某种能力（是否实现某个接口）
  • 类型保护函数
    • 参数：param
    • 返回值：param is interface
  • 接口可继承类
    • interface C extends A, B {}
    • 用A、B类的所有成员实现一个类型

5.索引器

  • [prop: string]: any
  • [prop: number]: any
    • 索引器需书写于类的最顶端（所有成员之前）
    • 可拓展对象
    • 除了类定义的成员外，可通过索引器进行成员拓展
    • 默认情况下，索引器可越过类型检查，需配置noImplicitAny
    • 在js中所有的成员名，包括数组索引都是字符串类型，成员名在使用数字类型时，会进行隐式类型转换
    • 在ts中成员名类型需要与索引器匹配

6.this指向约束

  • 全局直接调用
    • 严格模式：this指向undefined
    • 非严格模式：this指向全局对象
  • 对象方法调用
    • this指向对象本身
  • 事件处理函数调用
    • this指向事件处理对象
  • 箭头函数
    • this指向声明位置的this指向
  • bind、apply、call手动绑定this
    • this指向绑定值
  • 在ts中默认与js一致，严格this指向约束，需配置noImplicitThis
  • 在函数声明时，用第一个参数位置约定this指向

7.装饰器

  • 约束类
    • new () => object
    • ()内约束构造函数参数
  • 装饰类
    • 本质是一个函数
    • 参数：类本身
    • 返回值：
      • void：仅运行函数
      • class：替换原来的类
    • 运行时间
      • 定义类之后立即运行
      • 多个装饰器，按栈的结构顺序运行（新进后出）
  • 装饰成员
    • 本质是一个函数
    • 参数：
      • 参数1：
        • 静态成员：类本身
        • 实例成员：类原型
      • 参数2：成员名
      • 参数3：Object.defineProperty
    • 返回值：
      • void：仅运行函数
      • any：替换原来的成员
  • reflect-metadata
    • 第三方库
    • 用于保存元数据
  • class-validator
    • 第三方库
    • 用于验证
  • class-transformer
    • 第三方库
    • 用于将平面对象转换为类对象

（二）类型演算

1.关键字

  • typeof
  • keyof
  • in

2.预设

  • Partial 所有成员可选
  • Require 所有成员必填
  • Readonly 所有成员只读
  • Exclude<T, U> 剔除可以赋予给U的类型
  • Extract<T, U> 保留可以赋予给U的类型
  • NonNullable 剔除null、undefined
  • ReturnType 得到某个函数的返回类型
  • InstanceType 得到某个类的实例类型
